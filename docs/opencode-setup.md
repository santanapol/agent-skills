# OpenCode Setup (การติดตั้ง OpenCode)

คู่มือฉบับนี้จะบอกวิธีจับ Agent Skills มาเล่นคู่กับ OpenCode โดยให้มีประสบการณ์การใช้งานแทบจะกลืนเป็นเนื้อเดียวกับ Claude Code (ดึง skill ออกมาใช้เอง, สั่งขับเคลื่อนไลฟ์ไซเคิลอัตโนมัติ, และคุมกระบวนการได้เข้มงวดสุดๆ)

## Overview (ภาพรวม)

OpenCode รองรับการสร้างคำสั่งแบบปั้นเอง (custom `/commands`), แต่ดันไม่มีระบบปลั๊กอิน (native plugin system) ในตัวเครื่อง หรือระบบสวิตช์ทางเลือกสั่งโอน skills (automatic skill routing) ให้แบบ Claude Code

ดังนั้นเราจะมาเสกของพวกนี้แทน เพื่อสร้างความเท่าเทียมกัน:

- อัด system prompt เข้มๆ (ผ่าน `AGENTS.md`)
- ใช้เครื่องมือช่วย `skill` tool ที่แถมมาให้
- วางระบบจับเส้นทางส่อง skill ให้มาจากโฟลเดอร์ `/skills` แบบเป๊ะๆ

ทั้งหมดนี้จะก่อกำเนิดเป็น **agent-driven workflow (เวิร์กโฟลว์ชี้นำโดยเอเจนต์)** โดยที่ระบบจะหยิบ skills มาเลือกใช้แล้วสับเกียร์เดินหน้าทำเองแบบอัตโนมัติ

ถึงแม้ใน OpenCode เราจะสามารถปั้นคำสั่งเสกพวก `/spec`, `/plan`, และอื่นๆ ได้เองก็ตามเถอะ, แต่ชุดรวมร่างนี้เราจงใจเลือกหนทางใช้ agent สั่งการ (agent-driven approach) เพื่อมาทำงานแทน:

- ระบบจะจับสัมผัสอารมณ์จากจุดประสงค์ (intent) ของงาน แล้วดึงเอา skill ที่เหมาะสมขึ้นมาเอง
- ควบคุมมารยาทด้วยกฎเหล็กจาก `AGENTS.md`
- ไม่ต้องมานั่งกรอกพิมพ์สั่ง command ให้เมื่อยนิ้ว

วิธีแบบนี้มันจำลองพฤติกรรมในหน้างานของ Claude Code ได้เนียนตากว่ามาก เพราะเราไม่ต้องมามัวคอยป้อนเรียกให้ skill ตื่นมาทำงานเองเลย

---

## Installation (การติดตั้ง)

1. ก๊อปปี้โคลนโค้ดรหัสของคลังลงมาเลย (Clone the repository):

```bash
git clone https://github.com/addyosmani/agent-skills.git
```

2. เปิดโปรเจกต์นี้ใน OpenCode

3. ตรวจเช็คให้ชัวร์ว่ามีไฟล์พวกนี้สถิตอยู่ใน workspace ของคุณ:

- ไฟล์ `AGENTS.md` (อยู่หน้าบ้านสุด - root)
- โฟลเดอร์ `skills/` 

ติดตั้งเสร็จแล้ว! ไม่ต้องไปกดอะไรเพิ่มอีก

---

## How It Works (ระบบมันรันยังไง)

### 1. Skill Discovery (ระบบตรวจจับหา Skill)

Skills ทั้งหมดทั้งมวลจะพักพิงอยู่ที่:

```
skills/<ชื่อ-skill-แบบมีขีดกลาง>/SKILL.md
```

แก๊งค์ agents บน OpenCode ถูกสะกดจิตสั่งไว้ (ผ่าน `AGENTS.md`) ให้:

- ส่องหน้างานว่าควรงัดเอา skill ตัวไหนมาใช้
- เปิดประตูเรียกเครื่องมือ `skill` tool มาโชว์
- เดินตามกรอบกติกาของ skill ตัวนั้นๆ แบบเป๊ะๆ ไม่มีบิดเบือน

### 2. Automatic Skill Invocation (จับยัด Skill โหลดอัตโนมัติ)

ตัว agent จะประเมินจากคำขอของคุณ และจับโยงเข้ากับ skill ให้แบบออโต้

ตัวอย่างการจับคู่ (Examples):

- "สร้างฟีเจอร์นี้ที" → `incremental-implementation` + `test-driven-development`
- "วางระบบดีไซน์ให้หน่อย" → `spec-driven-development`
- "ช่วยแก้บั๊กนี้ที" → `debugging-and-error-recovery`
- "รีวิวโค้ดนี้ให้ด้วย" → `code-review-and-quality`

ผู้ใช้งาน **ไม่จำเป็นต้องเอ่ยปาก** บอกระบุชื่อ skill ตรงๆ เลยนะ

### 3. Lifecycle Mapping (Implicit Commands) (สั่งยกระดับวงจรชีวิต - แบบเนียนๆ)

วงจรชีวิตการพัฒนาระบบ (development lifecycle) ถูกผูกมัดร้อยเรียงกันไว้เบื้องหลังเรียบร้อย:

- กำหนดเป้า (DEFINE) → `spec-driven-development`
- วางแผนรบ (PLAN) → `planning-and-task-breakdown`
- สั่งลุยสร้าง (BUILD) → `incremental-implementation` + `test-driven-development`
- ส่องความเรียบร้อย (VERIFY) → `debugging-and-error-recovery`
- รีวิวงาน (REVIEW) → `code-review-and-quality`
- ปล่อยยานขึ้นดาว (SHIP) → `shipping-and-launch`

ไอ้ระบบพวกนี้มาแทนที่พวกคำสั่ง slash commands อย่าง `/spec`, `/plan`, และอื่นๆ แบบเนียนๆ

## Usage Examples (ตัวอย่างการเอาไปใช้)

### Example 1: Feature Development (ตัวอย่าง 1: งานสร้างฟีเจอร์)

ผู้ใช้ส่งคำขอ (User):
```
ช่วยเพิ่มระบบจัดการข้อมูลล็อกอิน (authentication) ในแอพนี้ให้ที
```

พฤติกรรมที่ agent แสดงออก:
- มันสัมผัสได้ว่านี่คืองานเสกของใหม่ (feature work)
- มันแอบไปปลุก `spec-driven-development`
- มันสาดสเปกงาน (spec) โชว์ออกมาก่อนที่จะดิ้นไปเขียนโค้ด
- มันพับแผนแล้วโยกย้ายเดินเครื่องปลุกทักษะวางแผน (planning) และการสร้างชิ้นงาน (implementation) ต่อไป

---

### Example 2: Bug Fix (ตัวอย่าง 2: งานซ่อมแซมแก้บั๊ก)

ผู้ใช้ส่งคำขอ (User):
```
เห้ย endpoint ตัวนี้มันสาด 500 errors รัวๆ เลย
```

พฤติกรรมที่ agent แสดงออก:
- มันกระชากคอ `debugging-and-error-recovery` ออกมาใช้งาน
- มันจำลองให้ตัวเองเห็นพัง (Reproduces) → ชี้เป้า(localizes) → ซ่อม (fixes) → เสริมเกราะป้องกันไม่ให้หน้าแตกซ้ำ (adds guards)

---

### Example 3: Code Review (ตัวอย่าง 3: งานรีวิวชิ้นโค้ด)

ผู้ใช้ส่งคำขอ (User):
```
รีวิวตัว PR นี้ให้ด้วยนะ
```

พฤติกรรมที่ agent แสดงออก:
- เรียกใช้งาน `code-review-and-quality` ด่วนจี๋
- จัดการกางตาข่ายรีวิวโครงสร้างเป็นระบบ (กาง 5 มิติ: correctness, design, readability, ฯลฯ)

---

## Agent Expectations (Critical) (ความคาดหวังที่มอบให้ Agent - สำคัญมาก)

เพื่อให้ OpenCode มันเดินหน้าโชว์ผลงานได้อย่างเต็มที่, ตัว agent จำเป็นต้องเชื่อฟังกฎระเบียบพวกนี้:

- คอยสังเกตส่องหน้างานเสมอว่า มี skill ตัวไหนที่เหมาะสมควรลากเข้ามาช่วยมั้ย
- ถ้าเจอว่า skill ตัวไหนเอามาใช้ได้ มันก็ MUST be used (ต้องหยิบมาใช้)
- ห้ามมักง่ายข้ามลำดับขั้นเวิร์กโฟลว์บังคับ (เช่น พวก spec, plan, test, ฯลฯ)
- อย่าทะลึ่งกระโดดไปลงมือตีโค้ดหน้าตาเฉย โดยยังไม่วางแผน

ซึ่งกฎเหล็กพวกนี้ถูกสลักฝังไว้ให้อย่างดีผ่านตัว `AGENTS.md` แล้ว

## Limitations (ข้อจำกัด)

- คำสั่งทางลัดแบบพิมพ์เอง (native slash commands) ใช้ไม่ได้ (แต่ไปสลับคุมบังเหียนใช้เป็นระบบจับพฤติกรรม intent mapping แทน)
- ไม่มีระบบดึงตัวปลั๊กอิน (plugin system) ในเครื่อง (ก็ทดแทนด้วยคำสั่ง prompt + รูปแบบโครงสร้าง)
- จะเสก skill ตัวไหนมาได้ มันขึ้นอยู่กับอารมณ์ความเชื่อฟังของตัวโมเดล(model compliance) เอง

แต่อย่างไรก็ดี, วิธีสับขาหลอกเหล่านี้ก็จำลองความเป็น Claude Code ออกมาได้คล้ายคลึงน่าทึ่งสุดๆ ไปเลยล่ะ

## Recommended Workflow (เวิร์กโฟลว์การทำงานที่อยากแนะนำ)

ปล่อยมันไหลไปตามธรรมชาติภาษาพูด (natural language) ของคนนี่แหละ:

- "ช่วยออกแบบสถาปัตยกรรมตัวฟีเจอร์นี้ให้หน่อย"
- "วางแผนหั่นงานให้ดูหน่อยสิ"
- "เริ่มสร้างชิ้นงานได้เลย"
- "ซ่อมแซมไอ้บั๊กเวรนี่ให้หน่อย"
- "ส่องรีวิวโค้ดหน่อย"

ตัว agent ของเรามันจะไปสแกนและจัดการป้อน skills ที่สอดคล้องกันมาเสิร์ฟให้คุณเองแบบอัตโนมัติ.

## Summary (สรุปทิ้งท้าย)

แพทเทิร์นระบบรวมร่างของ OpenCode จะขับเคลื่อนได้โดยเกิดจากการพึ่งพาของ 3 สิ่ง:

- โครงสร้างหน้าตา skills ที่เป็นระบบชัดเจน (ของที่อยู่ใน repo นี้)
- กฎระเบียบฝังหัวของ agent ที่แข็งแกร่ง (ผ่าน `AGENTS.md`)
- ระบบปลุกวิญญาณเรียก skill ให้ออโต้โดยอิงจากระบบความนึกคิด

ทั้งสามจะประสานพลังจนเกิดเป็น **ระบบสั่งงานให้ agent ไปลุยทำวิศวกรรมระดับงานโปรดักชันขั้นเทพ (fully agent-driven, production-grade engineering workflow)** โดยที่คุณไม่ต้องไปนั่งมุดหาปลั๊กอินมาลง หรือมานั่งกด manual commands เองเลยล่ะ
