# E2E Testing — End-to-End Testing

## 📌 คืออะไร
จำลอง Journey ของ User ตั้งแต่ต้นจนจบ Flow ครบวงจร โดยทดสอบผ่าน Browser จริงๆ แบบอัตโนมัติ

## 👥 ใครทำ
QA (Automated) หรือ SDET

## 🛠️ เครื่องมือที่แนะนำ (Zero Platform Standard)
- **`Playwright`** — เป็นตัวเลือกหลักสำหรับโปรเจกต์นี้ เนื่องจากมีจุดเด่นเรื่องความเร็วและการรองรับ **Multi-browser Context** ทำให้เราสามารถจำลอง Flow ที่ "แอดมินสร้างข้อมูล" และ "ผู้เล่นหน้าบ้านเห็นข้อมูล" ได้ใน Test ข้อเดียวโดยไม่ต้องเคลียร์ session

## 🔥 ความจำเป็น
**Must-have สำหรับโปรเจกต์สมัยใหม่** — ครอบคลุม Gap ที่ Unit Test และ Manual Test อาจพลาด
