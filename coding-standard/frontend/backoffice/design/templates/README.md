# Layout Templates Boilerplate

โฟลเดอร์นี้รวบรวมไฟล์คอมโพเนนต์เลย์เอาต์ต้นแบบ (Boilerplate Templates) ของระบบ Zero Platform Backoffice เพื่อให้นักพัฒนาสามารถคัดลอก (Copy-paste) หรืออ้างอิงไปใช้สร้างหน้าจอใหม่ได้อย่างเป็นสากลและรวดเร็ว

---

## 📂 โครงสร้างภายในโฟลเดอร์

```
templates/
├── PageContainer.tsx       # คอนเทนเนอร์ห่อหุ้มหน้าปกติ ( List / Dashboard )
├── DetailContainer.tsx     # คอนเทนเนอร์ห่อหุ้มหน้ารายละเอียดเชิงลึก (มีปุ่ม Back)
├── FiltersContainer.tsx    # บล็อกควบคุมตำแหน่งช่องค้นหาและตัวกรอง (Responsive Wrapping)
├── PageContentCard.tsx     # กล่องห่อหุ้มเนื้อหาหลัก (ตาราง, ฟอร์ม) แบบไร้ขอบสลับธีมอัตโนมัติ
├── index.ts                # จุดรวมศูนย์การ Export
│
├── ListTemplate.tsx        # [Boilerplate] หน้าแสดงตารางและตัวกรองข้อมูล
├── DetailTemplate.tsx      # [Boilerplate] หน้ารายละเอียดข้อมูลและแบบฟอร์มแก้ไข
├── DashboardTemplate.tsx   # [Boilerplate] หน้าแสดงสรุปสถิติและแผนภูมิภาพรวม
├── ResultTemplate.tsx      # [Boilerplate] หน้าแจ้งผลลัพธ์สำเร็จ หรือระบบขัดข้อง (403, 404, 500)
└── LayoutDemo.tsx          # ไฟล์รวมพรีวิวการทำงานจริงของทุกเลย์เอาต์
```

---

## 📑 รายละเอียดประเภทหน้าจอทั้ง 4 รูปแบบ

### 1. หน้าตารางรายการหลัก (List / Directory Pages)
*   **ไฟล์ต้นแบบ:** `ListTemplate.tsx`
*   **วัตถุประสงค์:** แสดงตารางข้อมูลหลักพร้อมตัวกรอง ค้นหา หรือปุ่มสร้างข้อมูลใหม่ (เช่น รายชื่อพนักงาน, รายการบิล, รายการตัวแทน)
*   **โครงสร้างหลัก:**
    *   คลุมหน้าจอชั้นนอกสุดด้วย `<PageContainer>`
    *   ใช้ `<PageContentCard>` เป็นกรอบห่อหุ้มข้อมูลตาราง
    *   วางเครื่องมือการค้นหาภายใต้ `<FiltersContainer>` โดยจำกัดขนาดของช่องค้นหาไม่ให้เกิน `300px` 
    *   ตาราง Ant Design (`Table`) ต้องเปิดใช้ `scroll={{ x: 'max-content' }}` เพื่อความยืดหยุ่นในการแสดงผล

### 2. หน้ารายละเอียดข้อมูลเฉพาะชิ้น (Detail / Form Pages)
*   **ไฟล์ต้นแบบ:** `DetailTemplate.tsx`
*   **วัตถุประสงค์:** แสดงเจาะจงข้อมูลเฉพาะรายการ หรือหน้าระบบลงทะเบียน/แก้ไขข้อมูลเชิงลึก (เช่น รายละเอียดบิลรายตัว, การตั้งค่าโปรไฟล์พนักงาน)
*   **โครงสร้างหลัก:**
    *   คลุมหน้าจอชั้นนอกสุดด้วย `<DetailContainer>` (จะได้รับฟังก์ชันและปุ่มย้อนกลับ `onBack` ไปยังหน้าหลักโดยอัตโนมัติ)
    *   บีบความกว้างหน้าจอเนื้อหาหลัก (Card / Form) ไม่ให้เกิน `720px` เพื่อให้อ่านข้อมูลได้สบายตา
    *   จัดแสดงผลข้อมูลแบบอ่านอย่างเดียวด้วย `<Descriptions>` แบบตีกรอบ (Bordered)

### 3. หน้าสรุปสถิติและแดชบอร์ด (Dashboard / Overview Pages)
*   **ไฟล์ต้นแบบ:** `DashboardTemplate.tsx`
*   **วัตถุประสงค์:** สรุปตัวเลขสถิติ ผลงานระบบ หรือแผนภูมิวิเคราะห์ภาพรวม (เช่น หน้าแรก Dashboard)
*   **โครงสร้างหลัก:**
    *   ใช้แถว `<Row gutter={[24, 24]}>` ในการเรียงกล่องสถิติ
    *   กล่องสถิติย่อย (`Statistic`) คลุมด้วย `<Card variant="borderless">` และปรับมุมมน `borderRadiusLG` ตามธีมหลัก
    *   แยกแผนภูมิวิเคราะห์หรือตารางความเคลื่อนไหวออกเป็นบล็อกสัดส่วนย่อยๆ ด้วยเลย์เอาต์ Grid

### 4. หน้าแสดงผลลัพธ์และข้อผิดพลาด (Error / Result Pages)
*   **ไฟล์ต้นแบบ:** `ResultTemplate.tsx`
*   **วัตถุประสงค์:** ใช้แจ้งผลการทำรายการสำเร็จ หรือนำทางเมื่อผู้ใช้เข้าไม่ถึงข้อมูล/ไม่พบหน้าเว็บ (เช่น หน้า 403 Forbidden, 404 Not Found, 500 Server Error)
*   **โครงสร้างหลัก:**
    *   จัดหน้าจอให้อยู่กึ่งกลางในพื้นที่ความสูงอย่างน้อย `80vh`
    *   ใช้งาน Ant Design `<Result>` เพื่อแสดงภาพสัญลักษณ์และข้อความอธิบายอย่างเหมาะสม พร้อมปุ่มกดนำทางกลับหน้าหลัก

## 🚀 วิธีการนำไปใช้งานในโปรเจกต์ใหม่

1.  คัดลอกไฟล์ Layout Wrappers (`PageContainer`, `DetailContainer`, `FiltersContainer`, `PageContentCard`, `index.ts`) ไปยังโฟลเดอร์คอมโพเนนต์ในแอปใหม่ของคุณ
2.  คัดลอกไฟล์ต้นแบบที่ต้องการ (เช่น `ListTemplate.tsx`) ไปสร้างเป็นหน้าใหม่ในโปรเจกต์
3.  เปลี่ยนชื่อฟังก์ชันและนำเข้า Layout Wrappers ผ่านพาธที่ถูกต้อง เช่น:
    ```typescript
    import { PageContainer, PageContentCard, FiltersContainer } from '../components/layout';
    ```
4.  สวมทับตาราง ฟอร์ม หรือสถิติลงในช่องแสดงผลหลักได้ทันทีค่ะ
